Generate an SDK from specifications

What is generated?

Let's have a look at the output of the generation process. The folder structure of the generated code source looks something like this:

  • src
    • api - contains classes that hold the operations described in the specification
    • helpers - contains helper functions that transform the data
    • models
      • base - contains generated interfaces based on the definitions described in the specification
      • core - contains manually created interfaces that extend the base models
      • custom - contains manually created custom interfaces used by the UI application, which are not provided by the specification

All the API model enhancements needed to implement the UI are in the helpers, core and custom folders, which of course contain hand-written code.
