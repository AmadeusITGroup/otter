Generate an SDK from specifications

Generator options

If a specification file includes dates, there are several ways to manage those with timezones depending on the use case of an application. We invite you to check out the documentation on how to manage dates in order to better understand the proposed solutions to manage dates.

Accordingly, there are multiple options for the generation of an SDK with dates involving the global property option stringifyDate. The value of this option can be set either in the generator command or in the configuration JSON file.

A reviver is a function whose purpose is to convert a JSON parsed object to the model the reviver is referring to (more information here.) Revivers are generated for SDKs that use:

  • non-stringified dates
  • dictionaries
  • extensible models

This means that there is no guarantee that revivers wil be generated. However, you may be in a case in which you want to be able to extend your SDK models and therefore ensure these functions are generated (more info on how to extend a model here).

To do this you can use the global property option allowModelExtension, which you can add either to the generator command or the configuration JSON file.
